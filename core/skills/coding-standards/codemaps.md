# Code Map: coding-standards/ Skill

## Overview

The **Coding Standards** skill provides universal coding standards, best practices, and patterns for **TypeScript, JavaScript, React, and Node.js** development.

## Purpose

This skill ensures:
- **Consistency**: Uniform code style across the codebase
- **Quality**: Best practices for maintainable code
- **Performance**: Patterns that optimize for speed and efficiency
- **Security**: Guidelines for writing secure code
- **Readability**: Standards that make code easy to understand

## Key Capabilities

| Capability | Description |
|------------|-------------|
| **TypeScript Standards** | Type definitions, interfaces, generics, strict mode |
| **JavaScript Patterns** | Modern ES6+ features, async/await, module patterns |
| **React Best Practices** | Component design, hooks, state management, performance |
| **Node.js Guidelines** | Server patterns, error handling, async operations |
| **Code Style** | Naming conventions, formatting, documentation |

## When to Use

Load this skill when:
- Writing or reviewing TypeScript/JavaScript code
- Creating React components
- Building Node.js applications
- Refactoring existing code
- Setting up linting/formatting rules
- Onboarding new developers to the codebase

## Usage

```
skill: {"name": "coding-standards"}
```

## Covered Technologies

- **TypeScript**: Type system, interfaces, generics, decorators
- **JavaScript**: ES6+, modules, async programming
- **React**: Functional components, hooks, context, performance optimization
- **Node.js**: Express, middleware, error handling, streams
- **Testing**: Jest, React Testing Library patterns
